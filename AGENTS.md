# SourceGen

C# incremental source generators for compile-time code generation.

## Principles (highest first)

1. **Correctness** ‚Äî Generated code must be semantically correct and compile-time validated.
2. **AOT compatibility** ‚Äî All generated code must pass Native AOT publishing without warnings.
3. **Performance** ‚Äî Generated code must be efficient and avoid unnecessary allocations.
4. **Incremental performance** ‚Äî Generators must be incremental (`IIncrementalGenerator`) and avoid allocations in hot paths.
5. **Developer experience** ‚Äî Clear diagnostics, minimal boilerplate, intuitive attribute API.

## Universal Rules

- Use C# 14 syntax ‚Äî see [csharp.instructions.md](.github/instructions/csharp.instructions.md)
- Read the relevant spec (`**/Spec/*.spec.md`) before implementing or modifying any feature
- Tests use TUnit: run with `dotnet run`, never `dotnet test --filter` ‚Äî see [csharp-tunit.instructions.md](.github/instructions/csharp-tunit.instructions.md)
- All generated code must include `// <auto-generated/>` header and `#nullable enable`
- All generated code must be deterministic ‚Äî same input produces identical output
- Generator data models must be immutable with value-based equality ‚Äî use `sealed record class` or `readonly record struct`
- Use `PolyType.Roslyn` utilities: `SourceWriter`, `ImmutableEquatableArray<T>`, `ImmutableEquatableDictionary<TKey, TValue>`, `ImmutableEquatableSet<T>`, etc.
- Generator pipeline: `ForAttributeWithMetadataName` ‚Üí `Select` (extract data model) ‚Üí `Collect` ‚Üí `RegisterSourceOutput`
- Keep diagnostic descriptors in the Analyzer project, not in the Generator
- If unsure whether an issue is a design decision or a test failure, **ask the user for clarification**

## Mandatory Final Steps

Every task that modifies code **MUST** complete these steps before finishing:

1. **Run Tests** ‚Äî Run all related tests via terminal. Fix any failing tests before proceeding.
2. **Review** ‚Äî Follow the workflow agent's review step.
