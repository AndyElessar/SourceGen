# SourceGen

C# incremental source generators for compile-time code generation.

## Principles (highest first)

1. **Correctness** — Generated code must be semantically correct and compile-time validated.
2. **AOT compatibility** — All generated code must pass Native AOT publishing without warnings.
3. **Performance** — Generated code must be efficient and avoid unnecessary allocations.
4. **Incremental performance** — Generators must be incremental (`IIncrementalGenerator`) and avoid allocations in hot paths.
5. **Developer experience** — Clear diagnostics, minimal boilerplate, intuitive attribute API.

## Universal Rules

- Use C# 14 syntax — see [csharp.instructions.md](.github/instructions/csharp.instructions.md)
- Read the relevant spec (`**/Spec/*.md`) before implementing or modifying any feature
- Tests use TUnit: run with `dotnet run`, never `dotnet test --filter` — see [csharp-tunit.instructions.md](.github/instructions/csharp-tunit.instructions.md)
- All generated code must include `// <auto-generated/>` header and `#nullable enable`
- All generated code must be deterministic — same input produces identical output
- Use `readonly record struct` or `sealed record class` for generator data models
- Use `PolyType.Roslyn` utilities: `SourceWriter`, `ImmutableEquatableArray<T>`, etc.
- Generator pipeline: `ForAttributeWithMetadataName` → `Select` (extract data model) → `Collect` → `RegisterSourceOutput`
- Keep diagnostic descriptors in the Analyzer project, not in the Generator

## Agent Topology

```text
Orchestrator (parent)
├── Explore         — read-only codebase research (always first)
├── Spec            — update specs under **/Spec/
├── Implement       — write code, run tests
├── Review          — read-only code review after every Implement
├── Doc (parent)    — write/update docs under docs/
│   ├── Explore
│   └── DocReview   — read-only docs review
└── DevOps (parent) — CI/CD workflows under .github/workflows/
    └── Explore
```

- **Parent agents** (Orchestrator, Doc, DevOps) manage `/memories/session/plan.md`
- **Child agents** (Implement, Review, DocReview, Spec) load plan before work; return `BLOCKED_*` if missing
- Protocol: [plan-memory-policy.instructions.md](.github/instructions/plan-memory-policy.instructions.md)

## Hierarchy

Subdirectory `AGENTS.md` files specialize these rules for their domain. Local rules inherit from this root and may add constraints but not relax them.
